# Markdown to Slide — Design Spec

## 개요

마크다운 파일을 발표용 HTML 슬라이드로 변환하는 Claude Code skill.
`/slides ./파일.md` 명령으로 실행하며, 입력 전처리 → 검증/보완 → 파싱 → HTML 생성으로 동작한다.
PDF, 이미지, PPTX 등 마크다운 외 입력도 지원한다.

## 요구사항

- **범용 목적**: 기술 발표, 사내 미팅, 강의 등 모든 상황에서 사용
- **단일 HTML 출력**: 외부 의존성 최소화 (highlight.js CDN만 사용)
- **챕터/슬라이드 2단계 구조**: `#`으로 챕터, `---`로 슬라이드 구분
- **미니멀 테마**: 흰 배경, 깔끔한 타이포그래피
- **변환 전 검증/보완**: 구조 검증 + 콘텐츠 개선 제안
- **다중 입력 포맷**: PDF, 이미지, PPTX → 마크다운 자동 변환 후 슬라이드 생성

## 마크다운 문법

### 기본 구조

```markdown
---
title: 발표 제목
author: 발표자 이름
---

# 1장. 소개

첫 번째 슬라이드 내용

> Note: 발표자 노트는 이렇게 작성합니다.

---

두 번째 슬라이드 내용

# 2장. 본론

세 번째 슬라이드 (새 챕터 시작)

---

네 번째 슬라이드
```

### 규칙

| 문법 | 역할 |
|------|------|
| `# 제목` | 챕터 구분 — 자동으로 챕터 타이틀 슬라이드 생성 |
| `---` | 같은 챕터 내 슬라이드 구분 |
| `> Note: ...` | 발표자 노트 (여러 줄 가능) |
| `![alt](path)` | 이미지 (로컬/웹 URL) |
| ` ```lang ` | 코드 블록 (구문 하이라이팅 + 라인 넘버) |
| YAML frontmatter | `title`, `author` 메타데이터 |

## 아키텍처

### 전체 흐름

```
[입력 파일] → [0. 전처리] → [1. 검증] → [2. 보완 제안] → [사용자 승인] → [3. 파싱] → [4. HTML 생성] → [HTML 파일]
```

### Phase 0 — 입력 전처리

입력이 `.md`가 아닌 경우, Claude의 내장 기능을 활용하여 마크다운으로 변환한다.

| 입력 포맷 | 처리 방법 |
|-----------|-----------|
| `.md` | 전처리 없이 바로 Phase 1로 진행 |
| `.pdf` | Read tool로 PDF 내용 추출 → 마크다운으로 변환 |
| `.pptx` | pptx skill로 읽기 → 마크다운으로 변환 |
| 이미지 (`.png`, `.jpg` 등) | Read tool로 이미지 인식 → 내용을 마크다운으로 변환 |

- 변환된 마크다운은 사용자에게 보여주고 확인 후 다음 단계 진행
- 별도 라이브러리 불필요 — Claude의 멀티모달/PDF 읽기 기능 활용

### Phase 1 — 검증

- 챕터(`#`) / 슬라이드(`---`) 구조가 올바른지
- 빈 슬라이드 감지
- 제목 누락 여부
- 코드 블록 언어 미지정 경고
- 이미지 경로 존재 여부 확인

### Phase 2 — 보완 제안

- 슬라이드당 내용 과다 시 분할 제안
- 핵심 포인트 요약/정리 제안
- 발표자 노트 자동 생성 제안
- 사용자에게 검증 결과 + 보완 제안을 보여주고 승인 후 진행

### Phase 3 — 파싱

- YAML frontmatter 파싱 (title, author)
- `#` 기준 챕터 분리
- `---` 기준 슬라이드 분리
- 각 슬라이드에서 `> Note:` 블록을 발표자 노트로 추출
- 마크다운 → HTML 변환

### Phase 4 — HTML 생성

- `template.html` 기반으로 단일 HTML 파일 생성
- CSS 인라인 포함
- 프레젠테이션 엔진 JS 인라인 포함
- highlight.js CDN 링크 포함
- 출력: 입력 파일과 같은 디렉토리에 `.html` 확장자로 저장

## HTML 슬라이드 구조

### DOM 구조

```html
<html>
<head>
  <style>/* 미니멀 테마 CSS */</style>
  <link rel="stylesheet" href="highlight.js CDN">
</head>
<body>
  <div class="progress-bar"></div>

  <div class="chapter" data-chapter="1">
    <div class="slide chapter-title">
      <h1>1장. 소개</h1>
    </div>
    <div class="slide">
      <div class="slide-content">...</div>
      <div class="speaker-notes">...</div>
    </div>
  </div>

  <div class="slide-number">3 / 12</div>

  <script>/* 프레젠테이션 엔진 */</script>
  <script src="highlight.js CDN"></script>
</body>
</html>
```

### 슬라이드 타입별 레이아웃

**챕터 타이틀 슬라이드:**
- `#` 헤딩에서 자동 생성
- 중앙 정렬, 큰 타이포그래피

**일반 콘텐츠 슬라이드:**
- 제목: 상단 좌측 배치
- 제목 아래 구분선
- 내용: 제목과 충분한 간격 확보, 좌측 정렬

**코드 슬라이드:**
- 제목: 상단 좌측 배치
- 제목 아래 구분선
- 코드 블록: highlight.js 구문 하이라이팅 + 라인 넘버

## 프레젠테이션 엔진

| 기능 | 키 | 동작 |
|------|-----|------|
| 슬라이드 이동 | `←` `→` | 이전/다음 슬라이드 |
| 챕터 이동 | `↑` `↓` | 이전/다음 챕터 |
| 처음/끝 | `Home` / `End` | 첫/마지막 슬라이드 |
| 전체화면 | `F` | 전체화면 토글 |
| 발표자 노트 | `N` | 노트 표시/숨김 토글 |
| 목차 오버레이 | `T` | 챕터 목록 오버레이 |
| 슬라이드 번호 | — | 우측 하단 `3 / 12` 상시 표시 |
| 프로그레스바 | — | 상단 가로 바 진행률 표시 |

## Skill 파일 구조

```
~/.claude/skills/markdown-to-slide/
├── skill.md              # skill 정의 (트리거: /slides, 워크플로우)
├── template.html         # 슬라이드 HTML 템플릿 (CSS + JS 포함)
└── examples/
    └── sample.md         # 예시 마크다운
```

### skill.md 역할

- `/slides ./파일.md` 로 트리거 (`.pdf`, `.pptx`, 이미지도 가능)
- Claude에게 워크플로우 지시:
  1. 입력 파일 읽기 (md가 아니면 전처리하여 마크다운 변환 → 사용자 확인)
  2. 검증 수행 (구조, 빈 슬라이드, 코드 블록 언어 등)
  3. 보완 제안 → 사용자 승인 대기
  4. `template.html` 기반 HTML 생성
  5. 같은 디렉토리에 `.html` 파일 출력

### template.html 역할

- 마크다운 내용이 채워질 HTML 뼈대
- 미니멀 테마 CSS 인라인
- 프레젠테이션 엔진 JS 인라인
- highlight.js CDN 링크

## v2 확장 계획 (향후)

| 항목 | v1 (현재) | v2 (향후) |
|------|-----------|-----------|
| 이미지 | `![](path)` 기본 지원 | 배경 이미지, 위치 지정 |
| 테마 | 미니멀 1종 | dark, colorful 등 다중 테마 |
| frontmatter | `title`, `author` | `theme`, `accent`, `font` 등 |

## 출력 규칙

- 입력: `./발표.md` → 출력: `./발표.html`
- 같은 디렉토리, 같은 파일명, 확장자만 변경
